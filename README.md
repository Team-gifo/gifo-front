### "인정을 담는 표현, 선물"

선물은 인류 역사와 궤를 같이해 온 진심의 매개체입니다. 단순히 물건을 주고받는 행위를 넘어, 서로의 인정을 확인하고 마음을 전하는 가장 따뜻한 소통 방식입니다.

디지털 시대의 선물은 그 어느 때보다 간편해졌지만, 역설적으로 대면의 온기와 '**기다림의 설렘**'은 점차 희미해지고 있습니다. 특히 품목을 미리 알고 받는 방식은 선물이 가진 본연의 즐거움인 '**의외성**'을 반감시키곤 합니다.

Gifo는 선물을 주고받는 기쁨을 뜻하는 '기뽀', '기뻐', '기쁘다' 에서 이름을 따왔습니다. 우리는 디지털 환경에서도 예상치 못한 순간의 즐거움과 설렘을 되살리고자 합니다.

### 1. 오직 한 사람만을 위한 서비스

Gifo는 불특정 다수에게 공개된 서비스가 아닙니다. '**고유 생성 코드**'를 보유한 사용자만 진입할 수 있는 독립된 시스템을 지향합니다. 이는 단순한 선물 전달을 넘어, '**나만을 위해 준비된 특별한 공간**'에 초대받는다는 심리적 가치를 부여합니다.

또한, 소중한 추억과 선물이 담긴 공간이 타인에게 노출되지 않도록 초대 코드 외에 '**2차 비밀번호**' 시스템을 도입하는 등 외부의 침해로부터 둘만의 공간을 더욱 안전하게 보호하여 프라이빗 서비스로서의 완성도를 높이고자 합니다. (현재 기능 개발 중)

<br>

### 2. 전달 받는 과정을 즐겁게

Gifo는 '**선물은 단순히 물건을 주고받는 행위**'라는 고착화된 틀을 깨고, 그 과정 자체를 하나의 즐거운 여정으로 바꿉니다.

- **액티브 콘텐츠**: 선물을 주고받는 모든 과정이 하나의 놀이가 될 수 있도록, 새롭고 창의적인 콘텐츠를 정기적으로 추가합니다.
  
- **콘텐츠 설계**: 추억 갤러리의 배치부터 각 콘텐츠의 노출 조건(퀴즈 정답 여부, 기회 횟수 등)까지, 보내는 이의 의도에 따라 세밀하게 조정할 수 있습니다.

<br>

### 3. 추억은 공유, 기억은 소장

Gifo는 정성스럽게 준비된 마음이 시간이 지나도 빛바래지 않도록, 기록의 영속성을 선물하고자 합니다.

- **이미지 다운로드**: 웹 페이지의 특성상 링크가 유실되거나 접속이 어려워질 상황을 대비하여, '추억 갤러리'의 프레임과 상대방이 준비한 선물 내역을 이미지 형태로 소장할 수 있는 기능을 제공합니다.
  
- **기프티콘 이미지**: 단순한 캡처가 아닌, 실제 선물(기프티콘) 형태의 프레임을 적용하여 시각적인 즐거움과 함께 언제 어디서든 다시 꺼내 볼 수 있는 '디지털 굿즈'로서의 가치를 더했습니다.

<br>

### 👥 페르소나
- 형식적인 축하에 지친 분
- 선물을 더욱 뜻깊게 전하고 싶은 분
- 형태에 구애받지 않는 선물(편지, 무형의 약속 등)을 전하고 싶은 분

<br>

### 💡 주요기능
- **선물 커스터마이징**: 활용 가능한 모든 요소를 주는 이가 직접 설계 + AI 추천 포장
- **프라이빗 시스템** : 고유 생성된 초대 코드나 다이렉트 전용 링크를 통해서만 접근할 수 있는 프라이빗 공간을 보장
- **추억 갤러리:** 소중한 사진과 텍스트를 담은 고유의 갤러리를 제공
- **액티브 콘텐츠:** 가챠(캡슐 뽑기), 퀴즈(객관식·주관식·OX) 등 받는 이의 참여를 유도하는 게임화(Gamification) 콘텐츠를 제공
- **공유하기** : 간편한 텍스트 복사 공유는 물론, 실제 선물을 받는 듯한 느낌을 주는 기프티콘 프레임 이미지 형태의 공유 방식을 지원

## 📁 프로젝트 구조 (Front-End)

```text
Gifo-Front/
├── assets/
│   ├── fonts/         # 커스텀 폰트 (PFStardust, WantedSans 등)
│   ├── images/        # 정적 이미지 및 로고 에셋
│   └── lottie/        # Lottie 애니메이션 에셋 (.json)
│
└── lib/
    ├── core/          # ⚙️ 전역 설정 및 유틸리티 (Clean Architecture)
    │   ├── constants/
    │   ├── di/        # Service Locator (GetIt) 설정을 통한 의존성 주입
    │   ├── router/    # GoRouter / ShellRoute 설정
    │   └── widgets/   # App 공통 UI 위젯 모음
    │
    ├── features/      # 📦 비즈니스 도메인별 모듈 (Feature 기반 분리)
    │   ├── addgift/   # [발신자] 8단계 맞춤형 선물 포장 컴포넌트
    │   ├── content/   # [수신자] 게임(가챠/퀴즈/언박싱) 및 전역 오디오
    │   ├── home/      # 서비스 랜딩 및 메인 화면
    │   └── lobby/     # [수신자] URL 진입 검증 및 로비, 갤러리 뷰
    │
    └── main.dart      # 앱 진입점
```

### 📚 주요 프론트엔드 라이브러리
<div align="center">

| 이름            | 설명                            | 목적 |
| -------------- | ----------------------------- | ----------------- |
| `flutter_bloc` | 상태관리 프레임워크                | 복잡한 비즈니스 로직과 UI 렌더링을 완전히 분리 |
| `go_router`    | 선언적 라우팅 패키지               | 프라이빗 URL 기반 파라미터 전달 및 ShellRoute 구현 |
| `freezed`      | 데이터 모델 생성 패턴               | 불변 객체(Immutable) 보장 및 JSON 직렬화 |
| `retrofit / dio`| REST API 통신 클라이언트         | 직관적이고 안정적인 서버 API 통신 인터페이스 구성 |
| `get_it`       | 의존성 주입(DI) 서비스 로케이터      | BLoC 및 Repository 패턴 간 결합도 완화 최소화 |
| `just_audio`   | 미디어 플레이어                   | 브라우저 정책(Autoplay)을 방어하는 수동 토글 BGM 연주 |
| `skeletonizer` | 로딩 UI 최적화 패키지              | 비동기 네트워크 이미지/데이터 통신 시 사용자 UX 개선 |
| `animated_text_kit` | 타이포그래피 애니메이션 패키지    | 결과창 등의 텍스트 타이핑 및 연출 극대화 |
